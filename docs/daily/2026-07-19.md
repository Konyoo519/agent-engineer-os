## 今天做了什么

共 4 Part,约 2 小时。

### Part 1 - Python Exception 基础 (40 min)

通过 6 个预测题学习异常机制:
1. divide(10,0) → 程序停止 + traceback 出现
2. try/except → try 块"全或无",一旦抛异常后续语句全跳过
3. raise → 手动抛异常,等价于系统自动抛
4. except 家族 → 异常是家族树,Exception 接所有后代
5. except 顺序 → 从上到下逐个匹配,先具体后宽泛
6. traceback → 从下往上读 = 出生地到没人接的崩溃点

### Part 2 - execute_tool 错误处理改造 (50 min)

设计 + 实现 + 验证。

**Schema 决策(用户主导)**:
- 成功 → `{tool, status: "success", content}`
- 失败 → `{tool, status: "error", category, message}`
- category(用户定义): `bad_args / not_found / wrong_type / timeout / unknown_tool / unknown_error`(6 类)

**实现**:
- `read_file_tool` 删掉内部 try/except(让它自然抛)
- 新增 `classify_error(e)` 查表函数(方法 B 字典)
- `execute_tool` 改返回 dict,内部 try/except + raise KeyError 复用 except 分支

**3 个 Case 验证(全通过)**:
- Case 1 读 tools.py → success → LLM 总结
- Case 2 读 missing_file.txt → not_found → LLM 立刻告诉用户
- Case 3 mock LLM 先给错参数再修正 → wrong_type → 自动恢复

**踩坑**:
- PowerShell Set-Content -Encoding UTF8 加 BOM(自己引回 Day 3 bug)
- Windows console GBK encoding 报 UnicodeEncodeError

### Part 3 - Pydantic Structured Output (30 min)

理论 + 应用到 Agent。

**学到的概念**:
- BaseModel = 自动验证的类,`__init__` 接收 kwargs 做字段验证
- `**` 解包机制(同 `func(**arguments)`)+ Pydantic 验证
- 嵌套 BaseModel 验证 + 字段路径错误信息(`args.path` 不是 "args")
- Pydantic 只查类型,不查业务(文件是否存在 Pydantic 不管)

**应用到 Agent**:
- tools.py 新增 `PrintArgs` / `ReadFileArgs`
- 新增 `TOOL_ARG_MODELS` 字典 + `validate_args` 函数
- agent.py 加 json.loads + Pydantic + execute_tool 三层错误拦截

### Part 4 - 端到端 5 轮错误恢复验证 (10 min)

`test_all_errors.py` 用 mock LLM 连续触发 5 轮:

| Round | 输入 | 拦截层 | 结果 |
|---|---|---|---|
| 1 | `{"path": 123}` | Pydantic 验证层 | 参数验证失败 |
| 2 | `{"path": "missing.txt"}` | execute_tool 运行层 | not_found |
| 3 | `{}` 空 JSON | json.loads 解析层 | JSON 解析失败 |
| 4 | `{"path": "tools.py"}` | 全部通过 | 成功 |
| 5 | 最终回答 | 跳出循环 | 完成 |

四层错误机制(json.loads / Pydantic / execute_tool / classify_error)互不替代,各管一段。
