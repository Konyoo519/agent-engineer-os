### Part 1 - Field 原理 (15 min)

`Field()` 在 Pydantic 中承担两件事, 而不是一件:

- **必填标记** (= `default=` 是否存在)
- **值约束** (= `min_length=1` / `ge=1` / `le=50` / `pattern=r"..."`)

4 行口头自测 (学生口头解释, 教师 Review):

| 字段定义 | is_required | 不传 | 传错 |
|---|---|---|---|
| `a: int` | True | 报 missing | 报 int_parsing |
| `a: int = Field(ge=1, le=50)` | True | 报 missing | 报 greater_than_equal / less_than_equal |
| `a: int = 10` | False | 用 10 | 报 int_parsing |
| `a: int = Field(default=10, ge=1, le=50)` | False | 用 10 | 报值越界 |

学生口头解释全部自洽 (教师实际跑了 `Demo(a=10.5)` 验证 `int_from_float` error type 真实存在).

### Part 2 - model_dump() 的本质 (15 min)

- BaseModel 实例是对象, 不是 dict —— 不能 `instance["key"]`、不能 `json.dumps(instance)`、不能 `func(**instance)`
- `model_dump()` = 对象 "拍扁" 回 dict 的网关
- 在 `validate_args` 里存在的原因: 给下游 `execute_tool(**arguments)` 喂 dict-friendly 数据
- 默认值会被 `model_dump()` 保留进 dict, 不会变 None

### Part 3 - TOOL_ARG_MODELS 复习 (10 min)

学生忘记了这个变量是什么. 教师重新讲清:

- 它不是函数, 是**全局字典**, 跟 `TOOLS` 对称
- `TOOLS[name]` —— 工具函数 (怎么执行)
- `TOOL_ARG_MODELS[name]` —— 参数校验类 (怎么校验)
- 加新工具时, 两个字典**都要**各加一行 (代码改动是 2 处, 不是 1 处)

### Part 4 - Optional + default 是两件事 (15 min)

`Optional[X]` 和 `default=Y` 在 Pydantic 里是独立的两个属性:

- `Optional[X]` —— 类型上接受 `None`
- `default=Y` —— 漏传时用 Y 兜底

4 种组合的真实行为 (教师用 `pydantic 2.13.4` 实际跑过):

| 字段定义 | 漏传 | 传 None | 传 10 |
|---|---|---|---|
| `a: Optional[int]` | 报 missing | 通过 | 通过 |
| `a: Optional[int] = 10` | 用 10 | 用 None | 用 10 |
| `a: int = 10` | 用 10 | 报类型错 | 用 10 |
| `a: int` | 报 missing | 报类型错 | 通过 |

`a: int` 不接受 None —— LLM 显式给 `null` 时直接拒.

### Part 5 - List[X] 与 None 是什么类型 (10 min)

- `List[X]` 字面意思 = X 的列表, 逐元素校验 (报错信息会带索引 `texts.1`)
- `None` 是 Python 的 "空" 标记, `type(None) == NoneType`, 对应 JSON 的 `null`
- `None` 不等于 0 / False / "", 它专门代表 "没有这个东西"
- `is None` 是 Python 判断 None 的标准写法

### Part 6 - Field 高级约束与教学风格更新 (10 min)

`Field` 常见约束归纳:

- 字符串: `min_length` / `max_length` / `pattern=r"..."`
- 数值: `ge` / `le` / `gt` / `lt`
- 列表: `min_length` / `max_length`
- 元数据: `description` / `default` / `frozen`

教学要点 (新规则, 见下方 "教学风格重大更新"):

- `pattern` 慎用 —— LLM 给的字符串经常带空格 / 中文, 过严的正则会误伤合法输入
- 约束按需加: 80% Agent 场景只需要 `min_length=1` 和数字的 `ge/le`

## 教学风格重大更新 (Day 5 起生效)

学生在 Part 1 反馈, 原话:

> "每次到了新概念的时候, 你先给我讲一下原理, 讲一下是什么东西.
>  不要立马给我一个新的名词, 新的函数, 新的代码, 我见都没见过, 然后让我预测."

**新规则**:

进入新概念时, 必须按下面顺序:

1. 原理 (为什么有这个东西?)
2. 字面定义 (这是什么东西?)
3. 真实代码例子 (我们在项目里会用到的形态)
4. 真实输出 (跑一遍给学生看)
5. 再让学生口头解释 / 预测

绝对**不要**在第 4 步之前让学生预测代码. 该规则作为 Day 6+ 默认教学风格.

## 今天真正掌握了什么

1. `Field()` 承担两件事: 必填标记 + 值约束 (元数据另外算)
2. required vs default 的本质 = 等号右边有没有写值
3. 类型错 (int_parsing) vs 值越界 (greater_than_equal) 是两种不同的 error type
4. `model_dump()` 是 BaseModel 实例 —— dict 的网关
5. `TOOL_ARG_MODELS` 是 "工具名字符串 —— 校验类" 查表字典
6. `Optional[T]` 跟 `default=Y` 是两件事, 分别处理 "接受 None" 和 "漏传兜底"
7. `List[X]` 字面 = X 的列表, 逐元素校验, 报错带索引
8. `None` 是 Python 的 "空" 标记, 等价 JSON `null`
9. "显式" = 在代码里明明白白写出来 (跟 "隐式" 相对)
