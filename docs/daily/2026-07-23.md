# 2026-07-23 学习日志 (Day 7)

## Learning Rule Improvement

今天发现：

第一次学习一个新的知识点时，直接预测运行结果，会导致学习效率下降。

因此新增永久教学规则：

所有新知识必须遵循：

原理

↓

字面

↓

真实代码

↓

真实运行结果

↓

总结规律

↓

预测

↓

练习

目的：

培养推理能力，而不是随机猜测代码。

## Tutorial 09 课程设计（Part 1 尚未开始教学）

### 今天实际完成了什么

Tutorial 08 已完成。今天开始共同设计 Tutorial 09，并将其命名为 `09_structured_agent`，同时创建了 Part 1 的教学材料和代码草稿。

Part 1 的主题是：**结构化 Agent action 与工具调用解析**。

Tutorial 08 使用普通文本协议：

```text
[TOOL:read_file:{"path":"tools.py"}]
```

Part 1 计划改用 JSON action 协议：

```json
{
  "kind": "tool_calls",
  "tool_calls": [
    {"name": "read_file", "arguments": {"path": "tools.py"}},
    {"name": "print", "arguments": {"text": "读取完成"}}
  ]
}
```

新增代码：

- `ToolCall`：表示一个待执行的工具调用
- `AgentAction`：表示 Agent 当前响应是工具调用还是最终回答
- `parse_action()`：把 JSON 文本解析成结构化对象
- `describe_action()`：观察结构化结果，不执行工具

### 代码草稿的运行结果（不计入学习完成）

```text
ACTION KIND: tool_calls
TOOL CALL COUNT: 2
CALL 1: read_file({'path': 'tools.py'})
CALL 2: print({'text': '读取完成'})
```

### 设计目标（不是学生已掌握内容）

Part 1 计划建立第一条新边界：

> 解析 Agent 想做什么，与执行工具，不是同一个职责。

代码草稿演示了一个响应可以被解析为多个独立的 `ToolCall`，但学生尚未开始学习和验证该规律。
